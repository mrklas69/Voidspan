4. Status barvy — kdy co použít

Status barvy (08–12) nejsou dekorativní. Každá má přesnou sémantiku. Nikdy nepoužívej status barvu na neutrální obsah — rozbiješ signalizaci.
ALERT · damaged, LOSS, fatal
CRITICAL · air < 10%, food < 5
WARNING · flotila přilétá, limit
OK · reserve (P2+ greenhouse)
INFO · dock, fleet, neutrální info

5. Progress bar — tři varianty

ASCII = nejlevnější (text), 8-bit segmentovaný = cinematic (HUD, klíčové resources), solid bevel = klasická arcade/RPG estetika. Doporučení dole.

5a. ASCII (v textu, task queue)

5b. 8-bit segmentovaný — task progress (amber fill)

5c. 8-bit segmentovaný — resource (threshold colors)

Barva se mění podle prahu — signalizace krize bez nutnosti číst číslo. Prahy: >50 % amber · 20–50 % warn-amber · 10–20 % warn-orange · <10 % alert.

5d. Solid bevel — arcade/RPG variant

jednodušší, bez segmentace

Doporučení

  • Task queue → 5a ASCII. Bar je inline v textu, font už sedí, nic extra.
  • HUD resources (AIR / FOOD) → 5c segmentovaný threshold. Primární status, barva = okamžitá signalizace krize.
  • Budoucí module HP / building progress → 5b amber segmentovaný. Monotónní akce, není třeba threshold signál.
  • 5d bevel — drží v záloze pro případ, kdy by segmenty byly vizuálně moc „pestrý".
