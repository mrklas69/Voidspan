## EventLog scroll — dva bugy najednou

User: „Nefunguje rolování v EventLog, zkontroluj všude."

**Bug 1:** `bg.on("wheel", (_p, _dx, _dy, dz) => ...)` četl **4. pozici = deltaZ** (prakticky vždy 0), místo 3. pozice = `deltaY`. Scroll tedy reálně nikdy nefungoval.

**Bug 2:** Bg-level wheel listener — pointer nad scrollbarem/Text objekty nedoručil event na bg pod nimi.

**Fix:** `this.scene.input.on("wheel", ...)` scene-level + bounds check na `computePosition() + panelW/panelH`. Izomorfismus s InfoPanel/ModulesPanel, které měly stejný pattern správně od začátku. Použití správného `deltaY` (4. arg na scene-level je deltaY).

## Event system redesign — verb = ikona, text = subjekt

User: „OK je nadbytečné. Zkus nějaký hezký systém zpráv."

**Princip:** Ikona nese akční sémantiku (start/pauza/dokončeno/rozpad/zásah/smrt), text drží jen konkrétní subjekt/parametr. Žádné opakování verba.

**Nové helpery v events.ts:**
- `eventIcon(ev)` — csq-aware lookup, přednostně csq→icon, fallback default VERB_CATALOG (TASK:START/RESUME → `>`, TASK:PAUSE → `||`)

**Před / Po:**
| Event | Před | Po |
|-------|------|-----|
| TASK:START | `# Zahájena oprava` | `> oprava` |
| TASK:PAUSE | `# Pozastavena oprava — low E` | `\|\| oprava — low E` |
| TASK:RESUME | `# Obnovena oprava` | `> oprava` |
| CMPL:OK | `OK Dokončena oprava` | `OK oprava` |
| ASSN | `» Přiřazen col_03` | `» col_03` |
| DECY:CRIT | `\ Zničeno rozpadem` | `\ zničeno` |
| DMG | `× Asteroid — ztráta 12 % HP` | `× Asteroid −12 % HP` |

**Další úpravy:**
- `TASK_VERB_CS` (infinitivy „Opravit") → `TASK_NOUN_CS` (ženské substantiva „oprava"). Gender agreement drží pro všechny FVP task kinds (repair/build/demolish).
- `taskLoc(task)` helper v world/task.ts — `moduleId ?? bay{idx}`. Konzumenty: ASSN + CMPL + TASK:PAUSE/RESUME/START (4× v protocol.ts). `[Kdy, Kde]` vyplněno všude, kde je lokace zřejmá.

## EventLog chip bug — ring buffer dirty gate

User: „První asteroid nepřidal nový filter DMG!"

**Root cause:** `renderBody()` měl dirty gate `events.length !== lastRenderedCount`, které selhává při nasycení ring bufferu (EVENT_LOG_CAPACITY=500). Push + shift zachovává length konstantní → nové verby se nedostanou do `seenVerbs` → chip se nevyrobí.

**Fix:** Zrušen length-gate, `seenVerbs` re-populate z bufferu každý frame (O(500) Set.add = μs). `rebuildChips()` má internal gate přes `seenVerbs.size` diff, takže DOM práce se děje jen při skutečné změně. `lastRenderedCount` pole odstraněno.
