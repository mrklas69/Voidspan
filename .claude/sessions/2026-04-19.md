## EventLog chip bug — ring buffer dirty gate

User: „První asteroid nepřidal nový filter DMG!"

**Root cause:** `renderBody()` měl dirty gate `events.length !== lastRenderedCount`, které selhává při nasycení ring bufferu (EVENT_LOG_CAPACITY=500). Push + shift zachovává length konstantní → nové verby se nedostanou do `seenVerbs` → chip se nevyrobí.

**Fix:** Zrušen length-gate, `seenVerbs` re-populate z bufferu každý frame (O(500) Set.add = μs). `rebuildChips()` má internal gate přes `seenVerbs.size` diff, takže DOM práce se děje jen při skutečné změně. `lastRenderedCount` pole odstraněno.

## Etapa 2 — server POC lokálně (`apps/server/`)

**Struktura:**
```
apps/server/
  package.json           (@voidspan/server, ws + tsx + @types/node + @types/ws)
  tsconfig.json          (Node ES2022, lib jen ES2022, types=["node"])
  src/
    server.ts            (~170 LOC — main loop, WS broadcast, graceful shutdown)
    persistence.ts       (~70 LOC — atomic save temp+rename, schema version check)
    smoke_client.ts      (~60 LOC — manual QA, exit po 2 snapshots)
  data/
    world.json           (auto-generated, 7.6 KB)
```

**Protocol (v `packages/shared/src/protocol.ts`):**
- `SCHEMA_VERSION = 1` konstanta — bump při breaking change → nuke persistence.
- **Server → Client:** `ServerHello` (full world + recentEvents[500]), `ServerSnapshot` (tick + world full dump), `ServerEvent` (single event mid-tick), `ServerPong` (keep-alive reply).
- **Client → Server:** `ClientPing` (keep-alive heartbeat).
- Žádný `SPEED` / `DECISION` v POC (Q4 — klient neposílá authoritative inputy). Přidá se v etapě 4 (klient migrace).

**Persistence (`persistence.ts`):**
- `saveWorld(w, path)` — atomic: `writeFileSync` do `.tmp` + `renameSync` (ext4/NTFS atomic swap). Wrapper `{ schemaVersion, tick, savedAt, world }`.
- `loadWorldOrNull(path)` — existence + JSON parse + schema version match. Při mismatch/corrupt/missing vrací null, server vytvoří fresh `createInitialWorld()`.
- Auto-mkdir parent directory (`./data/`) při prvním bootu.

**Server (`server.ts`):**
- Boot: `loadWorldOrNull(WORLD_PATH) ?? createInitialWorld()`.
- **Simulation loop:** `setInterval(stepWorld, TICK_MS=250)`. Před voláním zapamatuj `eventCountBefore = w.events.length`, po voláni broadcast nové eventy přes `w.events.slice(eventCountBefore)` jednotlivě.
- **SNAPSHOT frequency:** každých 10 ticků (= 2.5 s wall). Kompromis bandwidth (world ~50 KB JSON) vs. sync latency. Delta komprese P2+.
- **Persist loop:** `setInterval(saveWorld, 30_000)` paralelní s tick loop.
- **WebSocketServer:** port 3000 (override přes `VOIDSPAN_PORT` env). On `connection`: send `HELLO` immediately, add do `Set<WebSocket>`, listen na `PING` a `PONG`, cleanup on `close`.
- **Graceful shutdown:** SIGINT/SIGTERM → clearInterval × 2 → saveWorld → wss.close → exit. Fallback 3s timeout force exit.

**Dependencies install:** `pnpm install` proběhlo clean (13 nových balíčků: ws 8.18, tsx 4.19, @types/node 22.9, @types/ws 8.5, esbuild transitive). Workspace link `apps/server/node_modules/@voidspan/shared → packages/shared` funkční.

**Smoke test (`smoke_client.ts`):**
```
[smoke] connected
[smoke] HELLO schema=v1 tick=81 events=1
[smoke] PONG
[smoke] SNAPSHOT #1 tick=90 solids=90.00
[smoke] SNAPSHOT #2 tick=100 solids=90.00
[smoke] OK — hello=true pong=true snapshots=2 events=0
```

Stream funguje: HELLO hned po connect, PONG reply na PING, SNAPSHOT každých 10 ticků přesně.

**Persistence ověřeno:**
1. Server boot: `[persistence] ./data/world.json neexistuje — startuji fresh world`
2. Po 30 s: `[persist] tick=115 saved`, dále `tick=230 saved`
3. Kill + restart: `[persistence] načten snapshot tick=461, savedAt=2026-04-19T19:15:17.368Z` — svět pokračuje od ticku 461, ne od 0

### 1) Event.id + schema bump

**Testy:** 183/183 zelených (149 shared + 34 client, beze změny počtu). TS strict clean.

### 2) ws_client.ts (~180 LOC)

`apps/client/src/net/ws_client.ts` — tenký wrapper kolem browser WebSocket.

**Design:**
- `WsClient` třída + `WsClientHandlers { onHello, onSnapshot, onEvent, onStatus }` + `WsStatus = "connecting"|"connected"|"disconnected"|"reconnecting"`
- Reconnect: exp backoff 1s → 2s → 4s → 8s → 16s → cap 30s + jitter ±20 % (`Math.random()*2-1`)
- PING heartbeat každých 10 s (server timeout 30 s, margin)
- Backoff reset až po HELLO (ne po open) — open bez HELLO nic nedělá
- `closedByUser` flag blokuje auto-reconnect po manual `close()`
- `onError` jen logging — reconnect plánuje až `onClose` (jinak 2× call)
- PONG reply tiše drop — keep-alive, ne liveness probe

### 3) Feature flag v GameScene

- `new URLSearchParams(location.search).get("server")` detekce
- `this.serverMode: boolean` + `this.serverUrl: string | null` + `this.wsStatus: WsStatus` + `this.wsClient: WsClient | null`
- `createInitialWorld()` vytvoří placeholder i v server mode (panely čtou okamžitě, HELLO pak přepíše)
- `connectToServer(url)` — onHello/onSnapshot přepisuje `this.world` referencí (panely mají `getWorld = () => this.world` lazy closure)
- onEvent dedup: `if (events.last().id >= ev.id) return` + ring buffer splice
- `update()`: `if (!this.serverMode)` skip stepWorld
