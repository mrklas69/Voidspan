## Sezení 2 — Hosting & Stack

**Role:** technický architekt + kritik. Kritičnost 8/10.

### Hosting — rozhodnuto

Analýza Forpsi Easy Windows (sdílený IIS 10, PHP + .NET Core, MSSQL 2019, App Pool 256 MB / 25 % CPU, Timer pro plánované úlohy):

- Tři cesty zvažovány: (1) tick-based na Easy, (2) **VPS Linux Basic real-time**, (3) ASP.NET Core + SignalR na Easy.
- Volba: **A2 — VPS Linux Basic** (160 Kč/měsíc, 2 vCPU, 4 GB RAM, 40 GB NVMe, root). Cena VPS vs. vývojový strop Easy = jasná volba.
- Betelgeuse Easy zůstane pro doménu, hra pojede na VPS (A záznam přesměruje).

### Stack — rozhodnuto

| Vrstva | Volba |
|---|---|
| Runtime | **Node.js 22 + TypeScript** |
| Multiplayer framework | **Colyseus** (authoritative server, state diff sync, rooms) |
| Databáze | **SQLite** (prototyp) → **PostgreSQL 16** (produkce) |
| Frontend engine | **Phaser 3** + TypeScript |
| Reverse proxy / SSL | **Caddy** (auto HTTPS) |
| Deploy | **systemd service** + git pull |
| Monorepo | **pnpm workspace** (`apps/server`, `apps/client`, `packages/shared`) |

### Designový signál

Volba Phaseru místo vanilla SVG = Voidspan bude **živý animovaný svět** (pohybující se lodě, šířící se entropie, spektakulární fork event), ne statická mapa. Tohle držet v mysli při dalším designu.

### Poznámky / pasti

- Phaser: zapnout `pixelArt: true` a `setRoundPixels: true`, jinak anti-aliasing rozmlží 8-bit vzhled.
- SQLite + Colyseus: snapshotovat stav, ne psát každou změnu (write-lock).
- `packages/shared` = jediný zdroj pravdy pro enums, konstanty, Colyseus schémata (DRY).

### Otevřeno pro příště

- **Q17:** Rozsah prvního prototypu. Doporučeno P1 (monopoly mýto sim, 10 políček, 3 hráči, bez entropie/forků/NPC).
- Q7–Q11 (identita, fork mechanismus, atomická akce, NPC level, vítězná podmínka) stále otevřené.
- Otevření monorepa a Colyseus/Phaser skeletonu čeká na rozhodnutí Q17.

## Sezení 5 — Prostor a čas, datový model, Energy Model

**Role:** game designér + technický architekt + kritik. Kritičnost 8/10. Feedback Kudos!/Censure!.
**Fokus:** MINDMAP bod 3.1 (bagr mechaniky).

### Hlavní rozhodnutí

**Hierarchie entit světa** (nahrazuje roztříštěné pojmy):
```
WORLD → BELT → SEGMENT → MODULE → TILE
```
Pojem „Cell" retirován. Adresa `Teegarden.BeltN.SegXXX.MYY.TZ`.

**Konstanty:** `CONST_BELT_LENGTH = 256`, `CONST_SEGMENT_VOLUME = 16` (2×8 grid), `CONST_HABITAT_CAPACITY = 8`, `CONST_FOUNDING_CREW = 8`, `CONST_DAY_HOURS = 16`, `TIME_COMPRESSION ≈ 16×`, `CONST_MARSHAL_BASELINE = 4`.

**SHIP startovní konfigurace** — 2 segmenty (Bow + Stern). 8 kolonistů v kryo. Úvodní moduly: 2× SolarArray, Storage, 2× Habitat, MedCore, Assembler, CommandPost (s integrovanou Observatory), Engine→Dock. **Greenhouse CHYBÍ** = první stavební cíl. Narativní tlak: zásoby dojdou.

**Teegarden System** — soustava pojmenována podle reálné hvězdy Teegarden's Star (SO J025300.5+165258). Kolonie default = `Teegarden.Belt1`. Vertikální síť beltů (R1) → více orbitů, adresa `BeltN`.

### Energy Model (W / WD) — nová core mechanika

Jednotná mechanika pro **hmotnou práci** (Varianta A, úzký scope).
- `W` = okamžitý výkon v roli Build/Haul.
- `WD` = watt-day, jednotka práce.
- `duration_days = cost_WD / Σ(W_aktorů)`.
- Příklad demontáže Engine: 120 WD → 4 Constructor + 1 hráč = 48W → 2,5 dne hry (~2,5 h wall).
- Napájení omezuje počet simultánně aktivních aktorů (SolarArray 2×2 = 48 W = 4 Constructors + 1 hráč).

### Capability Matrix

Actor má 5 rolí (Build, Haul, Guard, Heal, Fight). **Specializace:** v jednom schedule slotu vykonává jednu roli. Matrix feeder do globálních VARS (lawlessness, healing_capacity, construction_rate…), které vstupují do triggerů / eventů / nákladů.

### Drone Fleet

Startovní pool SHIPu: **16 dronů** = 8 Constructors + 4 Haulers + 4 Marshals. Marshals = multi-funkční (police + IZS + admin), analogie Module Specialization → časem nahrazeno dedikovanou flotilou (Police, Medics, Firefighters, Judiciary AI, Admin AI).

**Lawlessness formula (KISS):**
```
belt.lawlessness = max(0, 1 - marshals_active / CONST_MARSHAL_BASELINE)
```
Jediná veličina s globálním dopadem na RULES beltu.

### Schedule & HP

- P1 aktivity: `Work | Eat | Sleep | Relax | Move`.
- Guard v P1 vykonávají Marshals, ne hráči.
- `HOMELESS` status — HP drain 1 / herní hodina. Oddálitelné kvalitní stravou, relaxem, léčením.
- Sit-out = brains preset; žádná imunita (poker analogie zavržena).

### Nové pozvánky (SCENARIO)

- **Founding Colonist Invitation** — pro prvních 8 hráčů, garantované oživení, Full Citizen start.
- **Capsule Invitation (Hail Mary)** — klasická nejistota revival / limbo / recycling.

### Observatory Event

Scripted event v SCENARIO §5.A. První detekce jiného beltu → přechod z izolace na R1 Belt Network. Integrovaná Observatory v CommandPost, později dedikovaný modul.

### Kudos z dnešního sezení (self)

- Ochrana před zavalením vlastními nápady (checkpoint pauza s nabídkou zápisu).
- Early detekce kolize dvou časových škál (live session vs. offline wall-clock).
