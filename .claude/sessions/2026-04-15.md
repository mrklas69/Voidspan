# Session 19 — UI Layer Stack axiom + Welcome dialog + průhlednost

Dlouhá designově-implementační noc 14.–15. 4. Hlavní téma: UI Layer Stack axiom (vrstevnatý z-order), Welcome Observer dialog s hero obrázkem (vygenerován, pak odstraněn), ladění průhlednosti boxů, drobné UX fixy.

## UI Layer Stack axiom (S19, hlavní téma)

Vrstevnatý z-order od dekorace k nejmodálnějšímu prvku. Izomorfní: výš = interaktivnější + naléhavější.

- **0 Hvězdné pozadí** (drift, layer 0)
- **1 Kosmické sprity** (asteroidy, kapsle, dopravní)
- **2 BELT** (segment, epicentrum — vertikální growth nahoru/dolů)
- **3 Ukotvené panely Top/Main/Bottom** (texty bez pozadí, zvažuje se průhledný panel)
- **3.5 Floating workspace** (K/U/Z/E/P — parkoviště, layer 4 odložen)
- **4 Link-triggered overlays** — kontextové, ne striktně hierarchické:
  - 4.1 Infotip (plaintext, autoclose)
  - 4.2 Karta (ASCII+barvy, autoclose)
  - 4.3 Popover (8bit wiki, autoclose)
- **5 Modální okna** (Close + ESC, **nezastavují čas**):
  - 5.1 info + Close (Help, Win, Loss, Mapa)
  - 5.2 velké přehledy (Výzkum, Obchod, Politika)
  - 5.3 Nastavení
  - 5.4 Chat s hráči, hlasování

**Axiomy:**
- ESC = globální bezpečný odchod (všechny vrstvy).
- `#hull-dark` jediné povolené pozadí overlay textů, mid/light varianty pro gradient 4.1→4.3.
- Texty ukotvených panelů bez pozadí.
- Modální nezastavují čas.

## Průhlednost boxů — Censure S19 + záhada

**Censure!** Pozadí dialogů bylo solid (nebo stejná barva jako canvas). User vyhlásil axiom: **všechny UI boxy mají transparentní pozadí, ladíme alpha per-box typ**. Memory zápis: `feedback_dialog_bg_transparent.md`.

### Bug: alpha vypadala binárně (0 nebo 100 %)
**Root cause:** `UI_PANEL_BG` = `COL_VOID_BLACK` = `#0a0a10` = stejná barva jako canvas backgroundColor. Alpha tedy neměla kontrast proti čemu projít — jediný viditelný rozdíl byl nad hvězdami. Fix: panel bg → `COL_HULL_DARK` (`#1a1e28`, slot 02).

### Border rect → stroke
Samostatný solid `border` rect pod bg zakrýval hvězdy. Fix: `setStrokeStyle(1, UI_BORDER_DIM)` přímo na bg rect.

### Ladění alpha
Několik iterací (0.88 → 1.0 → 0.4 → 0.2 → 0.5 → 0.7 → 0.8 → **0.9**). Finální hodnota: **panel alpha 0.9 (10% průhlednost), overlay 0.25**.

## Design brainstorming — Colony Goal & Status tree

Rozvažování „co je cílem hry/simulace, jak hodnotit stav kolonie". Výsledek:

### Colony Goal (jediný axiom)
- **GOAL:** trvale udržitelný život a rozvoj člověka.
- **Kompas:** hodně živých (a později šťastných) lidí s dobrou perspektivou.
- Goal **≠ win condition** — kompas, ne cílová čára. Konzistentní s „peak = příběh, ne score" (MINDMAP §1.2).

### Status tree — stav kolonie = posádka + základna
```
Status
 ├── I. Aktuální stav
 │   ├── I.1 Posádka  (kvantita + kvalita)
 │   └── I.2 Základna (kvantita + kvalita)
 ├── II. Udržitelnost (přežití)
 │   ├── II.1 Zásoby kolonistů (air/food/water runway + trend)
 │   └── II.2 Entropie základny (repair vs. decay)
 ├── III. Rozvoj                  [P2+ pahýl]
 └── IV. Společenský kapitál       [P2+ pahýl]
```
Agregace: **parent = worst child** (fraktální semafor). Prahy dashboard axiom 15/40 %.

### Simulation axiomy (GLOSSARY kánon)
- **Perpetual Observer Simulation** — svět žije nepřetržitě, bez hráčů/NPC, i s HP=0. Server end = jediný terminál.
- **Two Perspectives** — Observer nemá GAME_OVER; Player má (P2+). WIN/LOSS puzzle v P1 byl onboarding test, ne kánon.
- **Maslow axiom** — osy I–IV jsou nezávislé dimenze, ale strategie investic hierarchická (N na pevném N-1).
- **FVP** — First Viable Product ≠ P1 POC (ten byl puzzle). FVP = observable simulation sandbox, perpetual, automatika dronů, bez hráčů. NPC pak, hráči nakonec.

## Event Log System — kánon + spec

Navrženo na základě studia PocketStory (Agent report). 1:1 reuse i přizpůsobení retro terminálové estetice Voidspanu.

### Kánon (GLOSSARY §Event Log System)
- **Model-first axiom:** `World.events: Event[]` — jediný zdroj pravdy, ne Scene state.
- **Datový model:** `{tick, verb, csq?, loc?, actor?, item?, amount?, target?, text?, severity}`.
- **Verb katalog (FVP seed):** 23 čtyřznakových verbů s Unicode ikonami — BOOT `◉`, SPWN `+`, DEAD `†`, ARRV `↓`, DPRT `↑`, REPR `✓`, BLD `▲`, DEMO `▽`, DMG `×`, DECY `↘`, DRN `−`, PROD `*`, HAUL `→`, ASSN `»`, CMPL `✓✓`, FAIL `!`, IDLE `·`, WAKE `☆`, DOCK `⊙`, TICK `·`, STAT `§`, EVNT `◆`, SAY `"`, RPRT `»»`.
- **Consequence:** `OK / FAIL / PARTIAL / CRIT / START`.
- **Severity → paleta:** crit (ALERT_RED), warn (WARN_AMBER), pos (OK_GREEN), neutral (TEXT_DIM). Pure function `severity(verb, csq)`.
- **Ring buffer:** kapacita 500 (S20 rozhodnuto).
- **Lazy filter chips axiom:** chip pro verb se objeví až při prvním výskytu. TICK default off.
- **Žádná pauza simulace** — čas běží při otevřeném logu (Perpetual Observer axiom).
- **Click-through navigation** → IDEAS parkoviště (P2+).

### UI — Event Log Card (layer 3.5 floating)
- Hotkey **`[E]`** — v S20 zapsáno do Commands stringu, implementace v navazujícím úkolu.
- Pravý okraj, margin 12 px, šířka ~420, alpha 0.9, `COL_HULL_DARK`, stroke border.
- Font Jersey 25, size HINT 16 px.
- Header (title + chips + close `✕`), scrollable body, footer (count + clear).
- Auto-scroll na bottom, manuál pauzuje.

### Dopad UI
- Bottom Bar hint aktualizován: `[E] event log  [H] help  [WASD] select bay`.
- Nový font token **`FONT_SIZE_CMD = 12px`** — Commands řádek v Bottom Baru o ¼ menší než HINT (per user request).
- `GameScene.ts` použije `FONT_SIZE_CMD` místo `FONT_SIZE_HINT` pro hint ticker.
