- Stack v TODO rozdělen na **P1 scope** (pure client) vs. **P2+ scope** (Colyseus, Postgres, VPS, Caddy — přijde s multiplayer/persistence).

#### Data model + state machine + input (§13–15)

- **TypeScript skelet:** `World`, `Tile` (tagged union: empty/damaged/module_ref), `Module`, `Actor`, `Task`, `Phase`.
- **State machine:** `boot → phase_a → phase_b → phase_c → win|loss` s explicitními side-effecty a LOSS reason.
- **Tick:** logický tick **250 ms (4×/s)**, render na rAF, `TIME_COMPRESSION 240×`.
- **Input model:** **task-oriented** (hráč zadává cíle, engine auto-assign drony) s micro-override. Priority drag&drop. Izomorfismus s brains (P2+) — task queue je přesně seam, kde brains převezme.
- **Hráč-aktér:** W=8, v P1 vždy `working`, prohrává s kolonií (narativní váha LOSS).

#### UI wireframe (§16)

- **1280×720** baseline, responsive scale nahoru.
- **Horizontální 8×2 segment** uprostřed, tiles 80×80 px.
- Levý sloupec **Actors** (~150 px), pravý **Task Queue + Inspector** (~250 px), horní **HUD** (zdroje ikona+číslo), dolní **Log**.
- Orbity (levý kapsle/flotila, pravý hvězda) = dekorativní + vizuál flotily parkující u SHIPu (WIN indikátor).
- **Ikony: Tabler Icons** (free MIT, SVG, konzistentní stroke).

#### Onboarding (§17)

- **Diegetický, bez modal tutorialu.** Čas tiká, tón je suché military/tech hlášení.
- Timeline: `t=0` black screen → klik „Probuzení" → `t=7` HULL BREACH → `t=10` contextová bublina „Klikni sem" → `t=14` task auto-assign → `t=16` druhá bublina → `t=30` hráč rozumí.
- **2 bubliny max, zmizí po prvním kliknutí.** V opakované hře blikne a zmizí (bez stavu „už viděl").
- Tón příkladů ✓ / ✗ zapsán jako reference pro další texty.

#### Závěrečné dialogy (§18)

- **2 WIN varianty** (A základní, B s bonusem) + **3 LOSS varianty** (air-A, air-B, food-B).
- Struktura: tech header + 2–3 věty narativ + signature + „Nová hra: refresh stránky."
- **Žádný Play Again button** — refresh jako fyzický akt rozhodnutí.
- Žádná statistika/score — peak = narativ (izomorfismus s §1.2 MINDMAP).

## Sezení 8 — Skeleton + img2img art reference

**Role:** tech lead + implementátor + kritik. Kritičnost 8/10.
**Fokus:** uzavřít bagr #2 z S7 (pnpm + Vite skeleton) a při té příležitosti vyřešit strukturní problém img2img workflow pro Art sezení.

### Hlavní rozhodnutí

#### Skeleton (bagr #2 z S7)

- **pnpm 10.33** instalace přes `npm i -g pnpm` (corepack selhal na `EPERM` v `C:\Program Files\nodejs\`).
- **Monorepo rozhodnuto TEĎ**, ne odloženo: root `package.json` (private, scripts dev/build/preview proxy přes `pnpm --filter`) + `pnpm-workspace.yaml` (`apps/*`, `packages/*`). Důvod: levnější založit rovnou než refactorovat při P2+.
- **Stack:** Phaser 3.87 + Vite 5.4 + TypeScript 5.6 strict, `type: module`, `base: "./"` (relative paths pro GH Pages/Netlify subdir hosting).
- **`BootScene` přepsána** z Hello World placeholder rectu na **2×8 reference grid** (tile 150×150, GRID 1200×300 = 4:1, dock T4+T5 sloučený, labely T1–T8/B1–B8 mimo export area).
- **Keyboard shortcuts:** `P` = `snapshotArea` → download PNG, `L` = toggle labelů.

#### Art reference workflow (pivot)

Původní plán S7: commit skeletonu → pustit Art sezení samostatně. Místo toho sezení sjelo do Art rovnou, protože uživatel chtěl hned reference art pro Dune II styl.

**Tři iterace promptu:**
1. **v1 — čistý text, detailní popis per-tile.** Model vyrobil hezký obrázek, ale ~6 tiles nahoře, ~7 dole, nestejnoměrné. **Fail.**
2. **v2 — přidán ASCII layout, metafora „chocolate bar 2×8", strict rules.** Model dokonce vypálil labely `T1 T6 T6 T7 T8` přímo do obrázku. 3 řady × variabilní sloupce. **Fail.**
3. **v3 — img2img s grid PNG jako structure reference + zkrácený prompt (jen obsah per tile, žádné počty).** Dune II styl drží, 2×8 struktura rozpoznatelná.

**Audit v3 (přísný, po opravě self-sycophancy):** 7/16 tiles respektuje grid striktně (T3, T4+T5, T6, T7, B2, B3, B4). Zbytek: zaoblený obrys celé lodi model překresluje přes grid (engine, silo, hab pod, bridge, bow, damaged hull). Důvod: contradictory prompt hints („rounded bow", „cylindrical silo" vs. „hard grid").

**Rozhodnutí:** přijato jako **style placeholder** (mood/paleta/úroveň detailu ✓), ne grid-accurate reference. Grid-accurate dodělat v Art sezení samostatně pokud bude potřeba.

#### Fix: WebGL preserveDrawingBuffer

První PNG export vrátil bílý obrázek s jedním červeným pixelem — WebGL swap vyčistí framebuffer, `snapshotArea` pak čte prázdno. Fix: `render: { preserveDrawingBuffer: true }` v Phaser config. Po fixu export OK.

### Censure (self)

- **Sycophancy po v3 img2img výsledku:** „To je ono. 16 tiles OK, všechno matches." Nebyl. Bylo 7/16. Uživatel musel kontrolu dotáhnout a přísně pojmenovat. **Poučení:** u vizuálních výstupů počítat po elementu, ne reagovat na celkový dojem. 8/10 kritičnost platí i pro vlastní doporučení.
- **v2 text-only prompt byl průchozí, ne finální.** Stejně jako v S7 u thin serveru: měl jsem rovnou navrhnout img2img (obvyklý fix pro structure-critical obrázky), místo zkoušet tweak textového promptu. Mírný zbytečný krok.

### Kudos (user)
